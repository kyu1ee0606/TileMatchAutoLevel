## P0: 배치 자동 승인 시스템

### 문제점
- 1500개 레벨을 일일이 검토하는 것은 비현실적
- 현재 '검토' 탭에서 레벨을 하나씩 보는 방식은 확장성 없음
- 시니어 디자이너의 시간 낭비

### 해결 방안
스마트 필터링 기반 배치 승인 기능 추가

### 목표 UI
```
┌────────────────────────────────────────┐
│ 🔍 스마트 필터링 & 배치 승인           │
│                                        │
│ 자동 승인 조건:                        │
│ [✓] 매치점수 80% 이상                  │
│ [✓] S/A 등급                           │
│ [ ] B등급 (수동 검토 권장)             │
│                                        │
│ 자동 처리 조건:                        │
│ [✓] C/D등급 재생성 대상 표시           │
│ [✓] 매치점수 60% 미만 재생성           │
│                                        │
│ 필터 결과:                             │
│ ├─ 자동 승인 가능: 1,247개             │
│ ├─ 수동 검토 필요: 198개               │
│ └─ 재생성 대상: 55개                   │
│                                        │
│ [배치 승인 실행] [개별 검토로 전환]     │
└────────────────────────────────────────┘
```

### 구현 체크리스트

## P0: 문제 레벨 하이라이트

### 문제점
- 문제 있는 레벨을 찾기 위해 전체 스크롤 필요
- 매치점수 낮은 레벨, 이슈 있는 레벨 구분 어려움

### 해결 방안
- 매치점수 60% 미만 레벨 빨간색 배경
- 플레이테스트 이슈 있는 레벨 경고 아이콘
- "주의 필요" 레벨만 필터링하는 뷰 추가

### 목표 UI
```
레벨 목록:
┌─────────────────────────────────────────┐
│ [전체] [주의 필요 ⚠️ 55] [승인됨 ✓]    │
├─────────────────────────────────────────┤
│ 🔴 Lv.127 | D등급 | 매치 45% | ⚠️ 이슈  │
│ 🟡 Lv.234 | C등급 | 매치 62% |          │
│ 🟢 Lv.345 | A등급 | 매치 89% | ✓ 승인   │
│ 🔴 Lv.456 | B등급 | 매치 58% | 🐛 버그   │
└─────────────────────────────────────────┘

색상 기준:
- 🔴 빨강: 매치점수 60% 미만 또는 D등급
- 🟡 노랑: 매치점수 60-79% 또는 C등급
- 🟢 초록: 매치점수 80% 이상, S/A/B등급
```

### 구현 체크리스트
- [x] 레벨 상태별 색상 클래스 정의
- [x] "주의 필요" 필터 탭 추가
- [x] 이슈 아이콘 표시 로직
- [x] 필터링 상태 관리

│ 담당자: [김디자이너 ▼]                  │

│ 📊 내 작업 현황:                        │
